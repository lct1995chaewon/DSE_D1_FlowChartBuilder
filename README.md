## 🌟 項目簡介

**FlowLearn** 是一個基於 HTML5/JavaScript 的單文件應用程序，靈感來自經典教學軟件 Raptor。它旨在幫助學生（特別是中學電腦科學生）直觀地理解程式設計的三大基本結構：
1.  **序列 (Sequence)**
2.  **選擇 (Selection / IF-ELSE)**
3.  **迴圈 (Iteration / Loop)**

用戶可以透過拖放節點繪畫流程圖，並即時「執行」該流程圖，觀察變數變化、輸入輸出及邏輯分支，從而將抽象的代碼邏輯具象化。

---

## ✨ 核心功能

*   🎨 **直觀繪圖**：拖曳式介面，支援開始、結束、處理、輸入、輸出、判斷六種標準節點。
*   ⚡ **即時執行**：一鍵運行或逐步除錯，節點高亮顯示當前執行步驟。
*   📊 **變數監察**：右側面板實時顯示所有變數的數值變化。
*   🧠 **智能語法**：支援中英文混合輸入（如 `AND`, `&&`, `及`），降低入門門檻。
*   📖 **內置範例**：一鍵載入序列、IF、IF-ELSE、迴圈等經典教學案例。
*   🚀 **零依賴**：單一 HTML 文件，無需伺服器、Node.js 或任何外部庫，離線可用。

## 🛠 使用指南

### 1. 建立流程圖
*   **新增節點**：點擊頂部工具列的按鈕（如 `🔷 處理`），然後在畫布空白處點擊。
*   **編輯內容**：雙擊節點或使用右側屬性面板輸入指令。
*   **連接節點**：從節點的圓點（Port）拖曳線條到另一個節點。
    *   *注意*：菱形（判斷）節點有「是 (右)」和「否 (左)」兩個分支。

### 2. 執行與除錯
*   **▶ 執行**：自動跑完整個流程。
*   **⏭ 逐步**：每按一次執行一個步驟，適合課堂講解。
*   **🔄 重置**：清空變數與日誌，重新開始。

## 📝 語法參考 (Syntax Guide)

FlowLearn 使用簡化的自然語言與程式碼混合語法。

| 節點類型 | 語法格式 | 範例 |
| :--- | :--- | :--- |
| **📥 輸入 (Input)** | `輸入 [變數]` 或 `INPUT [變數]` | `輸入 score`<br>`INPUT age` |
| **🔷 處理 (Process)** | `[變數] = [運算式]` | `total = price * qty`<br>`count = count + 1`<br>`sum = 0; i = 1` (多行用分號) |
| **📤 輸出 (Output)** | `輸出 [內容]` 或 `OUTPUT [內容]` | `輸出 total`<br>`輸出 "及格"` (文字需加引號) |
| **🔶 判斷 (Decision)** | `[條件表達式]` | `score >= 50`<br>`age > 18 AND hasMoney == 1`<br>`rain == 1 OR snow == 1` |

### 運算符支援
*   **算術**：`+`, `-`, `*`, `/`, `%` (餘數)
*   **比較**：`>`, `<`, `>=`, `<=`, `==` (等於), `!=` (不等於)
*   **邏輯**：
    *   且：`AND`, `&&`, `及`
    *   或：`OR`, `||`, `或`

## 🎓 內置教學範例

點擊介面上的 **📖 示範例子** 即可載入：

1.  **序列結構**：計算矩形面積 (`Area = Length * Width`)。
2.  **IF 單分支**：及格檢查 (分數 >= 50 ? 輸出 "及格")。
3.  **IF-ELSE 雙分支**：帶傘判斷 (下雨 ? "帶傘" : "不用帶傘")。
4.  **迴圈結構**：累加總和 (計算 1 + 2 + ... + N)。

## ⚠️ 常見問題 (FAQ)

**Q: 為什麼判斷節點報錯「表達式錯誤」？**
> A: 請檢查變數名是否已在之前的「輸入」或「處理」節點中定義。變數名稱**大小寫敏感** (例如 `Score` 和 `score` 是不同的)。

**Q: 如何輸出文字？**
> A: 文字必須使用雙引號包圍，例如 `輸出 "Hello"`。如果沒加引號，系統會將其視為變數名。

**Q: 程式陷入無限迴圈怎麼辦？**
> A: 系統設有安全機制，超過 10,000 步會自動停止。請檢查迴圈內是否有改變條件變數的步驟（如 `i = i + 1`）。
