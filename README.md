FlowLearn 流程圖學習工具 - 使用指南
📖 項目簡介
FlowLearn 是一個專為初學者（特別是中學電腦科學生）設計的網頁版流程圖編寫與執行工具。它直觀地展示了程式設計的三大基本結構：序列 (Sequence)、選擇 (Selection) 和 迴圈 (Iteration)。

用戶可以透過拖放節點、連接線條來繪畫流程圖，並即時「執行」該流程圖，觀察變數變化與輸出結果，從而理解程式邏輯。

🚀 快速開始
1. 啟動方式
將提供的 HTML 代碼保存為 .html 文件（例如 flowlearn.html）。
使用現代瀏覽器（Chrome, Edge, Firefox, Safari）直接打開該文件即可運行。無需安裝任何伺服器或軟體。
2. 介面概覽
頂部工具列：選擇節點類型（開始、結束、處理、輸入、輸出、判斷）及操作工具（連接、刪除）。
中央畫布：繪畫流程圖的工作區域，支援拖曳移動節點。
右側面板：
節點屬性：編輯選中節點的內容。
變量監察：即時顯示程式執行時的變數數值。
輸出主控台：顯示執行日誌、錯誤訊息及最終輸出結果。
🛠 核心功能操作
1. 建立節點
在頂部工具列點擊所需的節點按鈕（如 🔷 處理）。
在中央畫布的空白處點擊，即可放置該節點。
預設內容：新節點會帶有預設文字（如 x = 0），請務必雙擊修改。
2. 編輯內容
雙擊任意節點，彈出編輯視窗。
輸入對應的邏輯指令（詳見下文「語法規則」）。
點擊「確定」保存。
3. 連接節點
點擊工具列的 🔗 連接 按鈕。
點擊起始節點的連接埠（小圓點），再點擊目標節點的連接埠。
注意：
判斷節點 (菱形) 有兩個輸出端口：右側代表「是 (Yes)」，左側代表「否 (No)」。首次連接時系統會詢問你該連線屬於哪一個分支。
其他節點通常由底部連向頂部。
4. 移動與刪除
移動：直接拖曳節點即可調整位置，連線會自動跟隨更新。
刪除：
方法 A：點擊工具列 🗑 刪除，然後點擊想刪除的節點。
方法 B：選中節點後按鍵盤上的 Delete 鍵（若瀏覽器支援）。
5. 執行流程圖
▶ 執行：從「開始」節點一路自動運行到「結束」節點。過程中節點會發光顯示當前步驟。
⏭ 逐步：每按一次執行一個步驟，適合除錯或詳細講解。
🔄 重置：清除所有變數與主控台紀錄，準備重新執行。
📖 示範例子：一鍵載入預設的教学範例（序列、IF、IF-ELSE、迴圈）。
📝 語法規則 (Syntax Guide)
FlowLearn 使用簡化的自然語言與程式碼混合語法，方便學生理解。

1. 輸入節點 (Input)
用於接收用戶輸入並儲存到變數。

格式：輸入 [變數名] 或 INPUT [變數名]
範例：
輸入 score
輸入 age
INPUT name
2. 處理節點 (Process)
用於計算或賦值。

格式：[變數名] = [運算式]
支援運算：+, -, *, /, % (餘數)
範例：
total = price * qty
count = count + 1
area = length * width
sum = sum + i (累加)
3. 輸出節點 (Output)
用於顯示結果。

格式：輸出 [內容] 或 OUTPUT [內容]
內容可以是：變數名、文字串（需用雙引號 "" 包圍）、或運算式。
範例：
輸出 total (顯示變數值)
輸出 "你好" (顯示文字)
輸出 "面積是：" + area (混合顯示，視具體實作而定，建議分開輸出)
OUTPUT "及格"
4. 判斷節點 (Decision)
用於條件分支，結果必須為 True 或 False。

比較運算符：>, <, >=, <=, == (等於), != (不等於)
邏輯運算符：
且：AND, &&, 及
或：OR, ||, 或
範例：
score >= 50
age > 18 AND hasMoney == 1
rain == 1 OR snow == 1
x != 0
🎓 教學範例說明
點擊介面上的 📖 示範例子 可載入以下經典案例：

1. 序列 (Sequence) - 計算矩形面積
邏輯：依序輸入長、寬，計算面積，最後輸出。
重點：理解程式由上而下執行的順序。
2. IF 選擇 (Single Selection) - 及格檢查
邏輯：輸入分數，若 >= 50 則輸出「及格」，否則直接結束。
重點：理解條件成立才執行特定動作。
3. IF-ELSE 選擇 (Double Selection) - 帶傘判斷
邏輯：輸入天氣代碼 (1=雨, 0=晴)。若為 1 輸出「帶傘」，否則輸出「不用帶傘」。
重點：理解二選一的邏輯分支。
4. 迴圈 (Iteration) - 累加總和
邏輯：輸入數字 N，利用迴圈計算 1 加到 N 的總和。
重點：理解如何利用判斷節點形成迴圈結構。
⚠️ 常見問題與注意事項
變數名稱錯誤：
確保「輸入」、「處理」、「判斷」、「輸出」中使用的變數名稱完全一致（大小寫敏感）。
錯誤範例：輸入用 Score，判斷用 score。
無限迴圈：
如果在迴圈中忘記改變條件變數（例如忘記寫 i = i + 1），程式會執行超過 10,000 步後強制停止並報錯。
連線斷開：
每個節點（除了結束節點）都必須有向外的連線，否則執行到該處會停止。
判斷節點的「是」與「否」兩條路徑都必須連接後續節點（可以是不同的節點，也可以是同一個結束節點）。
特殊字符：
文字輸出請務必使用雙引號 ""，否則系統會嘗試將其當作變數名稱處理而報錯。
💡 給教師的建議
視覺化除錯：利用「逐步執行」功能，配合右側的「變量監察」面板，讓學生清楚看到每一步變數數值的改變。
生活化情境：鼓勵學生將數學題目改為生活情境（如：買飲料折扣、遊戲闖關條件），降低對程式碼的恐懼感。
錯誤模擬：故意製造一個沒有連接「否」路徑的流程圖，讓學生觀察程式行為，加深對「完整邏輯」的理解。
